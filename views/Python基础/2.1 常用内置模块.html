        %a 本地（locale）简化星期名称

        %H 一天中的第几个小时（24小时制，00 - 23）
        %I 第几个小时（12小时制，01 - 12）
        %j 一年中的第几天（001 - 366）
        %m 月份（01 - 12）
        %M 分钟数（00 - 59）
        %p 本地am或者pm的相应符
        %S 秒（01 - 61）
        %U 一年中的星期数。（00 - 53星期天是一个星期的开始。）第一个星期天之前的所有天数都放在第0周。
        %w 一个星期中的第几天（0 - 6，0是星期天）
        %W 和%U基本相同，不同的是%W以星期一为一个星期的开始。
        %x 本地相应日期
        %X 本地相应时间
        %y 去掉世纪的年份（00 - 99）
        %Y 完整的年份
        %Z 时区的名字（如果不存在为空字符）

        # 五、subprocess 模块

        subprocess模块主要用于创建子进程，并连接它们的输入、输出和错误管道，获取它们的返回状态。

        通俗地说就是通过这个模块，你可以在Python的代码里执行操作系统级别的命令，比如“ip a”、“du -sh”等等。subprocess模块替代了一些老的模块和函数，比如：

        os.system
os.spawn*
        1
        2
        • subprocess过去版本中的call()，check_call()和check_output()已经被run()方法取代了。
        • run()方法为3.5版本新增。
        • 推荐使用run()方法调用子进程，执行操作系统命令
        • 在更高级的使用场景，你还可以使用Popen接口
        subprocess.run(args, *, stdin=None, input=None, stdout=None, stderr=None, shell=False, timeout=None, check=False, encoding=None, errors=None)
        1

        run()方法的返回值，表示一个进程结束了。CompletedProcess类有下面这些属性：

        • args 启动进程的参数，通常是个列表或字符串。

        • returncode 进程结束状态返回码。0表示成功状态。

        • stdout 获取子进程的stdout。通常为bytes类型序列，None表示没有捕获值。如果你在调用run()方法时，设置了参数stderr=subprocess.STDOUT，则错误信息会和stdout一起输出，此时stderr的值是None。

        • stderr 获取子进程的错误信息。通常为bytes类型序列，None表示没有捕获值。

        • check_returncode() 用于检查返回码。如果返回状态码不为零，弹出CalledProcessError异常。

        • subprocess.DEVNULL 一个特殊值，用于传递给stdout、stdin和stderr参数。表示使用os.devnull作为参数值。

        • subprocess.PIPE 管道，可传递给stdout、stdin和stderr参数。

        • subprocess.STDOUT 特殊值，可传递给stderr参数，表示stdout和stderr合并输出。

        run 方法调用方式返回 CompletedProcess 实例，和直接 Popen 差不多，实现是一样的，实际也是调用 Popen，与 Popen 构造函数大致相同，例如:

        # Popen() 方法

        Popen 是 subprocess的核心，子进程的创建和管理都靠它处理。

        class subprocess.Popen(args, bufsize=-1, executable=None, stdin=None, stdout=None, stderr=None, 
preexec_fn=None, close_fds=True, shell=False, cwd=None, env=None, universal_newlines=False, 
startupinfo=None, creationflags=0,restore_signals=True, start_new_session=False, pass_fds=(),
*, encoding=None, errors=None)
        1
        2
        3
        4

        常用参数：

        • args：shell命令，可以是字符串或者序列类型（如：list，元组）
        • bufsize：缓冲区大小。当创建标准流的管道对象时使用，默认-1。 0：不使用缓冲区 1：表示行缓冲，仅当universal_newlines=True时可用，也就是文本模式 正数：表示缓冲区大小 负数：表示使用系统默认的缓冲区大小。
        • stdin, stdout, stderr：分别表示程序的标准输入、输出、错误句柄
        • preexec_fn：只在 Unix 平台下有效，用于指定一个可执行对象（callable object），它将在子进程运行之前被调用
        • shell：如果该参数为 True，将通过操作系统的 shell 执行指定的命令。
        • cwd：用于设置子进程的当前目录。
        • env：用于指定子进程的环境变量。如果 env = None，子进程的环境变量将从父进程中继承。

        # Popen 对象方法

        • poll(): 检查进程是否终止，如果终止返回 returncode，否则返回 None。
        • wait(timeout): 等待子进程终止。
        • communicate(input,timeout): 和子进程交互，发送和读取数据。
        • send_signal(singnal): 发送信号到子进程 。
        • terminate(): 停止子进程,也就是发送SIGTERM信号到子进程。
        • kill(): 杀死子进程。发送 SIGKILL 信号到子进程。

        简单实例：

        #!/usr/bin/python3

# -*-coding:utf-8 -*-
# @File    : sub_popen.py
# @User    : kame


import time
import subprocess


def shell_cmd(command):
    sub_popen = subprocess.Popen(command, shell=True, stdout=subprocess.PIPE, stderr=subprocess.PIPE, encoding="utf-8")
    sub_popen.wait(2)
    if sub_popen.poll() == 0:
        # communicate 输出为列表
        print(sub_popen.communicate()[0])
    else:
        print("执行命令失败")


shell_cmd("python3 --version")

        # 打印磁盘情况
#!/usr/bin/python3

# -*-coding:utf-8 -*-
# @File    : pa.py.py
# @User    : kame


import paramiko

ssh = paramiko.SSHClient()

# 将信任的主机加到 host_allow 列表
ssh.set_missing_host_key_policy(paramiko.AutoAddPolicy())
# 连接服务器 ssh.connect("服务器IP地址", "服务器端口号", "用户名", "密码")
ssh.connect("172.16.249.101", "22", "root", "www.123.com")

stdin, stdout, stderr = ssh.exec_command('df -hl')

print(stdout.read().decode('utf-8'))

print(stdout.read().decode())

ssh.close()

        # 服务器文件修改内容

        sftp 对象可以在线修改远程服务器上文件的内容

        #!/usr/bin/python3

# -*-coding:utf-8 -*-
# @File    : paramiko_sftp_file.py
# @User    : kame

import paramiko

# 登录远程服务器
client = paramiko.SSHClient()

client.set_missing_host_key_policy(paramiko.AutoAddPolicy())
# client.connect("服务器IP地址",,"服务器端口号","用户名", "密码", timeout=5)
client.connect("172.16.249.101", "22", "root", "www.123.com", timeout=5)

sftp = client.open_sftp()

# 远程服务器文件地址为/root/tmp.log
remoteFile = sftp.open("/root/tmp.log", 'a')
remoteFile.write("\n");
remoteFile.write("追加网址  www.liuwq.com");
remoteFile.write("\n");
# 关闭操作文件对象
remoteFile.close()
# 关闭 sftp连接对象
sftp.close()

        # 查询文件

        使用 sftp 对象获取远程服务器上的文件列表

        #!/usr/bin/python3

# -*-coding:utf-8 -*-
# @File    : paramiko_sftp_select.py
# @User    : kame


import paramiko

client = paramiko.SSHClient()
client.set_missing_host_key_policy(paramiko.AutoAddPolicy())
# client.connect("服务器IP地址",,"服务器端口号","用户名", "密码", timeout=5)
client.connect("172.16.249.101", "22", "root", "www.123.com", timeout=5)

sftp = client.open_sftp()

# 远程服务器 目录地址  /root
for fileName in sftp.listdir("/root"):
    # endswith 判断结尾
    if fileName.endswith(".log"):
        print(fileName)
# 关闭sftp
sftp.close()
